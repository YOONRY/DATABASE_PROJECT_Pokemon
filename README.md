# Pokémon Encounter Database Web

Flask + SQLite 기반 포켓몬 출현 정보 웹 서비스

---

## 📌 프로젝트 개요

본 프로젝트는 **데이터베이스를 중심으로 한 웹 응용 서비스**를 직접 설계·구현하여  
DB의 **저장, 조회, 검색 역할**을 실습적으로 이해하는 것을 목표로 한다.

1세대 포켓몬 게임(포켓몬 레드, 그린, 블루, 피카츄)을 기반으로  
**지역별 포켓몬 출현 정보**를 조회할 수 있는 웹 서비스를 제작하였다.

---

## 🎯 서비스 목적

- 포켓몬 게임 내 **지역별 등장 포켓몬 정보**를 직관적으로 제공
- SQL을 이용한 실제 데이터 설계 및 조회 로직 경험
- 데이터 베이스 구축 경험 
---

## 🗂 데이터베이스 구성

### 주요 테이블

#### 1. `pokemon`
| 컬럼명 | 설명 |
|------|------|
| Pid | 포켓몬 ID |
| Pname | 포켓몬 이름 |
| type1 | 주 타입 |
| type2 | 부 타입 (nullable) |

#### 2. `location`
| 컬럼명 | 설명 |
|------|------|
| Lid | 지역 ID |
| Lname | 지역 이름 |

#### 3. `encounter`
| 컬럼명 | 설명 |
|------|------|
| Eid | 인카운터 ID |
| pokemon_id | 포켓몬 ID (FK) |
| location_id | 지역 ID (FK) |
| min_level | 최소 레벨 |
| max_level | 최대 레벨 |

---

## 🔍 데이터 출처

- 포켓몬 1세대 기준
- **인게임 데이터를 직접 수작업으로 정리하여 데이터베이스에 입력**
- 등장 지역, 레벨 범위 등은 게임 정보 기반

---

## 🧩 주요 기능

### 1️⃣ 지역 목록 조회
- 좌측 사이드바에 모든 지역 목록 표시
- 클릭 시 해당 지역의 포켓몬 등장 정보 표시

### 2️⃣ 지역별 포켓몬 출현 정보
- 포켓몬 이름
- 타입(색상으로 구분)
- 등장 레벨 범위
- 포켓몬 스프라이트 이미지 표시

### 3️⃣ 포켓몬 이름 검색
- 포켓몬 이름을 입력하면
- 해당 포켓몬이 **출현하는 지역 목록**을 반환

### 4️⃣ 다크 모드
- 버튼 클릭으로 다크/라이트 모드 전환
- 사용자 설정을 `localStorage`에 저장

---

## 🖥 서비스 화면 구성

### 메인 페이지
- 상단: 서비스 제목 + 다크모드 버튼
- 좌측: 지역 목록
- 우측: 지역 이미지 + 등장 포켓몬 목록

### 사용자 입력
- 포켓몬 이름 검색 입력창

---

## 🔄 데이터베이스 사용 방식

- Flask API에서 SQL 쿼리를 통해 데이터 조회
- 프론트엔드는 `fetch()`를 통해 JSON 데이터 수신
- 사용자 입력(검색어, 지역 선택)에 따라
  **동적으로 SQL 쿼리 결과를 반영**
